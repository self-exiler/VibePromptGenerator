**软件开发需求文档生成工具技术规格说明书**

**1. 总体要求**
本工具旨在通过结构化的参数配置界面，引导用户输入关键信息，并自动生成格式统一的Markdown文档，作为规范的软件开发需求说明书。

**2. 技术栈**

- 使用纯前端（HTML和JAVASCRIPT）完成。

**3. 功能需求**

**3.1 选项卡一：需求配置**

1. **语言选择框**：下拉列表，支持选择开发语言，如 C++、Python、C# 等。
2. **Wheel（轮子）选择框**：下拉列表，根据所选语言动态加载对应的可选轮子。轮子可多选。
3. **软件架构选择框**：下拉列表，提供 BS架构、CS架构、桌面应用、命令行工具等选项。
4. **运行与调试环境选择框**：下拉列表，提供如 Linux-Bash、Windows-PowerShell、Windows-CMD 等环境选项。
5. **总体需求描述框**：多行文本输入框，用于填写软件总体要求。
6. **模块需求编辑区**：
   - 采用标签页形式组织，用户可通过按钮动态添加或删除标签页。
   - 每个标签页包含以下字段：
     - 模块名称（单行文本框）
     - 前端/上层需求（多行文本框）
     - 后端/底层需求（多行文本框）
   - 在“前端/上层需求”与“后端/底层需求”文本框旁，各设有一个“常用语句”按钮。点击后弹出选择窗口，以多选框形式列出预置常用语句，用户可多选，确认后所选内容将以换行形式插入对应文本框。
7. **其他注意事项输入框**：多行文本输入框，用于填写开发与执行过程中的其他说明。
8. **生成 Prompt.md 按钮**：点击后弹出文件保存对话框，用户指定路径与文件名，工具将生成的Markdown文档输出至该位置。通过config.json记忆上一次保存位置。
9. **重置按钮**：清除当前选项卡内的所有输入内容。

**3.2 选项卡二：语言与轮子管理**

- 提供界面用于管理开发语言及其对应轮子的列表，支持增、删、改操作。

**3.3 选项卡三：常用语句管理**

- 提供界面分别管理“前端/上层需求”与“后端/底层需求”的常用语句库，支持增、删、改操作。
